dumbpkg - my stupid package manager

dumbpkg is a very basic package manager (you could really say it isn't a package manager at all! You might say it is a software aggregator.) for *NIX. It is solely written in bash and has very few dependencies and does not require sudo.
It installs files to ~/.dumbpkg/installed

Installation

When it wants to be, installation is pretty simple.
To install, simply type or copy-paste: curl https://raw.githubusercontent.com/tomblcode/dumbpkg/master/setup.sh | bash into your shell.

Dependencies

dumbpkg only relies on git and curl

Tested

Tested on a MacOS 10.12 iMac, a Raspberry Pi running Raspbian and a bunch of Ubuntu versions and flavours.

Commands

You can try dumbpkg help to see an updated version of this:
Do dumbpkg install PACKAGENAME or dumbpkg i PACKAGENAME to install stuff.
Do dumbpkg remove PACKAGENAME or dumbpkg r PACKAGENAME to uninstall stuff.
Do dumbpkg search PACKAGENAME or dumbpkg s PACKAGENAME to search for stuff.
Do dumbpkg create PACKAGENAME to create packages that you could let anyone install with a pull request.
Do dumbpkg info PACKAGENAME to get the information stored in the data files about packages.
Do dumbpkg list to list all insallable packages.
Do dumbpkg config to edit your config file. (~/.dumbpkg/config.sh)
Do dumbpkg update to update the packages list to be able to install newer packages.
